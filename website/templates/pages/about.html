Core Features

üè¢ Multi-Tenant Architecture

Genes uses a clone-based isolation system that makes building SaaS platforms and multi-site applications straightforward. Each "clone" is a completely isolated tenant with its own data, automatically filtered by clone_id. 

// Create a tenant
g::run("db.createClone", array("hash" => "client-abc", "name" => "ABC Corp"));

// Set active tenant
g::run("db.setClone", "client-abc");

// All queries automatically filtered
$items = g::run("db.getItems"); // Only client-abc items
$users = g::run("db.getPersons"); // Only client-abc users

üóÑÔ∏è Flexible Data Model

The 5-table schema (clones, persons, items, labels, events) is generic enough for most applications yet specific enough to be immediately useful. Each table stores JSON data, giving you schema flexibility when you need it. 

// Tables automatically created with g::run("db.createSchema")
clones   - Master tenant table
persons  - Users, contacts, profiles (has clone_id)
items    - Content, products, entities (has clone_id)
labels   - Categories, tags, groups (has clone_id)
events   - Activity log, history (has clone_id)

üîå Database Agnostic

Use MySQL, SQLite, or no database at all. Config.json for simple data storage, HTML partials for static sites, or mix-and-match as needed. 

// MySQL
g::run("db.connect", array("driver" => "mysql", "database" => "mydb", ...));

// SQLite
g::run("db.connect", array("driver" => "sqlite", "database" => "app.db"));

// No database - use config.json, HTML partials, or any other data source

üé® Template System

Simple variable substitution in HTML partials. No complex templating language to learn, just {{variable}} placeholders that get replaced with your data. 

// Load partial with variables
function loadPartial($file, $vars = array()) {
    $content = file_get_contents("templates/" . $file);
    foreach ($vars as $key => $value) {
        $content = str_replace("{{" . $key . "}}", $value, $content);
    }
    return $content;
}

echo loadPartial("page.html", array("title" => "Welcome", "user" => "John"));

Why "Genes"?

üß¨

Like DNA carries the blueprint for life, Genes Framework carries the essential building blocks for web applications. The clone-based architecture mirrors biological cloning - each tenant is a perfect copy of the base structure, yet completely independent and unique.

Just as genes are simple building blocks that combine to create complex organisms, Genes Framework provides simple, reusable components that combine to build sophisticated applications.
